漏洞

在del_note函数中：

free(*((void **)notelist[v1] + 1));
free(notelist[v1]);
puts("Success");

在删除的时候，只是单纯进行了free，而没有设置为NULL，那么显然，这里是存在 Use After Free 的情况的。

此时如果我们再申请一个note2,大小为0x8的话,根据malloc的分配原则 ，它会将最近free的堆块优先分配，它先会给note分配8个字节的空间来存放指针，这个堆块是note1 free后的堆块，接着它会将note0存放指针的堆块分配给note2的content，这个时候如果我们将note0的指针覆盖成别的函数地址，再通过print_note调用的话就可以劫持函数执行流了。

为了更加高效地利用 fast bin，glibc 采用单向链表对其中的每个 bin 进行组织，并且每个 bin 采取 LIFO 策略，最近释放的 chunk 会更早地被分配，所以会更加适合于局部性。也就是说，当用户需要的 chunk 的大小小于 fastbin 的最大大小时， ptmalloc 会首先判断 fastbin 中相应的 bin 中是否有对应大小的空闲块，如果有的话，就会直接从这个 bin 中获取 chunk。如果没有的话，ptmalloc 才会做接下来的一系列操作。

解题思路

  • 申请 note0，real content size 为 16（大小与 note 大小所在的 bin 不一样即可）
  • 申请 note1，real content size 为 16（大小与 note 大小所在的 bin 不一样即可）
  • 释放 note0
  • 释放 note1
  • 此时，大小为 16 的 fast bin chunk 中链表为 note1->note0
  • 申请 note2，并且设置 real content 的大小为 8，那么根据堆的分配规则
  • note2 其实会分配 note1 对应的内存块。
  • real content 对应的 chunk 其实是 note0。
  • 如果我们这时候向 note2 real content 的 chunk 部分写入 magic 的地址，那么由于我们没有 note0 为 NULL。当我们再次尝试输出 note0 的时候，程序就会调用 magic 函数。

利用思路

  1. 设置author name长度为32，33位的\x00被book1地址覆写后，输出author name即可泄露book1地址。
  2. 通过修改author name，使后两位变为00，布置使我们的book1的指针（变00时）指向的是book1的description。
  3. 通过修改book1的description，使description内容为b1_1。
  4. b1_1中的book name和description指针，指向book2的name。
  5. 输出book1，那么book1的description就是b1_1，就可以打印出book2的name的地址，实现泄露，得到libc_base。
  6. 将book2的description设置为__free_hook函数，将boo2的name设置为system("/bin/sh")函数，再free boo2，调用__free_hook，执行system("/bin/sh")。

思路

按顺序PUT 1，2，5，3，4，x（防止被top chunk合并）

依次释放掉 5, 3, 1, 之后利用 `del` 输入时候的 `getline`，将 3 填满，造成 `off-by-one`
将 4 `free` 掉进行合并（伪造 `prev_size`），构成`overlap`
分配 1 的大小，使得 libc 地址被写到 2 里，就可以泄露出地址，然后将 5 分配出来，写入需要的内容，就可以 fastbin attack 了。

from pwn import *

p = process('./bcloud')
elf = ELF('./bcloud')
libc = ELF('/lib/i386-linux-gnu/libc.so.6')

def cmd(ch):
    p.sendlineafter('>>',str(ch))

def add(size,con):
    cmd(1)
    p.sendlineafter('content:',str(size))
    p.sendlineafter('content:',con)

def edit(idx,con):
    cmd(3)
    p.sendlineafter('id:',str(idx))
    p.sendlineafter('content:',con)

def delete(idx):
    cmd(4)
    p.sendlineafter('id:',str(idx))

p.sendline('\xff\xff\xff\xff')
topchunk_addr = heap_base + 0xd8
