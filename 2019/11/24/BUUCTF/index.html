babyfengshui

分析程序

通过add函数可以发现它构造了两个相邻堆块，分别为desc（size自己输入）和结构体（0x80）申请。

再看一下update函数，发现有一个防止堆溢出的保护(v3 + *ptr[a1]) < ptr[a1] - 4，翻译一下就是user.desc + text len < user.name，应该是想防止desc堆溢出，但这只适用于两个堆相邻的情况

题目思路

先申请两个堆块，在申请一个较小的堆块存放/bin/sh

然后释放掉0，在申请一块大小为size0+0x80的堆块

这样我们就可以绕过update函数那个保护，修改struct1的desc指针为free@got，泄露libc，最后在free@got中写入system，delete(2)就相当于执行system('/bin/sh')

ciscn_s_3

分析程序

一开始看到这道题的时候一点思路都没有……后来看了wp，知道了还有一个叫SROP的技巧。我们可以看到程序执行了read(0,&buf,0x400)和write(1,&buf,0x30)。
所以可以通过read把/bin/sh写到栈中，然后可以通过write函数leak栈的地址。

发现gadgets函数把rax设置成15，也就是sigreturn的系统调用号（64位）

然后的思路就是：

  1. 设置edi为/bin/sh的地址
  2. 设置rax为execve的调用号
  3. 造成系统中断，执行execve(“/bin/sh”)

至于怎么leak栈的地址，可以通过argv[0]。argv[0]是一个栈地址，指向的是程序的名称，也就是rbp的值，一般都保存在rsp上面。我们可以得到argv[0]的值，计算其与参数的偏移，从而得到参数的地址。
如我们输入字符串aaa：

0x7fffffffde08-0x7fffffffdcf0=0x118
