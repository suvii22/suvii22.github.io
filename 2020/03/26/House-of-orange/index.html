修改 top chunk 的size

size 需满足以下条件：

  • top chunk 大于 MINSIZE，即 0x10
  • old_size < nb + MINSIZE
  • PREV_INUSE = 1
  • old_top + old_size 页对齐

申请一块大内存来触发 sysmalloc() 中的 _int_free()

_int_free() 之后，top_chunk 会进入 unsorted bin 中，我们申请一个小于此大小的 chunk 后，因为 bk 指针会保留，所以可以 leak libc ，之后可以进一步 leak 堆地址（这里没有搞明白，之后再来填坑）。

unsorted bin attack

我们可以通过 unsorted bin attack 来覆盖 IO_list_all 指针；覆盖完 IO_list_all 为 main_arena+88，由于我们无法完全控制 main_arena 的内容，所以需要 chain 来转移到下一个 IO_FILE。

chain 位于_IO_FILE_plus 偏移为 0x68 的位置，所以我们要把 main_arena+88+0x68 的地方覆盖为我们可控制的区域（top_chunk）。我们可以把 old_top 置为0x61，这样触发一个不合适的 small chunk 分配，old_top 就会被放到 smallbins[5] 中，smallbins[5] 刚好在 main_arena+88+0x68 的位置，这样 fp 指向了old_top 。

申请内存，触发 glibc 异常处理

我们要伪造内存，将 _IO_OVERFLOW 修改为 system，fp 开头8字节赋值成 “/bin/sh\x00”来 get shell 。为了触发 _IO_OVERFLOW ，我们需要满足一些条件：

      if (((fp->_mode <= 0 && fp->_IO_write_ptr > fp->_IO_write_base)
#if defined _LIBC || defined _GLIBCPP_USE_WCHAR_T
       || (_IO_vtable_offset (fp) == 0
           && fp->_mode > 0 && (fp->_wide_data->_IO_write_ptr
                    > fp->_wide_data->_IO_write_base))
#endif
       )
      && _IO_OVERFLOW (fp, EOF) == EOF)   <--- system('/bin/sh')

  • fp->_mode <= 0

  • fp->_IO_write_ptr > fp->_IO_write_base
