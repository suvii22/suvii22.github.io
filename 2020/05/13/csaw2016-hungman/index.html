分析

这个程序一开始要输入name，name存储在堆中：

//相关数据体
struct 
{
    int score;
    int name_len;
    char *name;
}

然后让你玩游戏，游戏是是让你猜随机生成和你name等长的随机字符串，有三次猜错机会，全部猜出来会获得一次change name的机会。

if ( v3 == 'y' )
      {
        s = malloc(0xF8uLL);
        memset(s, 0, 0xF8uLL);
        v8 = read(0, s, 0xF8uLL);
        *(_DWORD *)(a1 + 4) = v8;
        v14 = strchr((const char *)s, 10);
        if ( v14 )
          *v14 = 0;
        memcpy(*(void **)(a1 + 8), s, v8);
        free(s);
      }

我们可以看到新输入的name是写到存储着原来name堆块中的，于是这里便可以造成堆溢出，修改结构体中的指向name指针：

pwndbg> x/40gx 0x603000
0x603000:    0x0000000000000000    0x0000000000000031
0x603010:    0x6161616161616161    0x6161616161616161
0x603020:    0x6161616161616161    0x0000000000006161
0x603030:    0x0000000000000000    0x0000000000000091
0x603040:    0x0000001b00000000    0x0000000000603010   <---cover here
0x603050:    0x0000000000000000    0x0000000000000000

于是我们可以：

  • 第一次change name，修改指针为strchr@got，同时leak libc
  • 第二次change name，覆写stchr为system
  • 第三次change name，输入/bin/sh

如果我们一开始较长的name，那么buf中所有字母基本就都会出现，依次把所有字母输一圈就可以赢得游戏获得改名字的机会
